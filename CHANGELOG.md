# 更新日志

## 3.5.4 (2026-05-21)

### 功能修复

- 修复 **未登录时空跑** 问题：之前 cookie 在但服务端 session 已过期，或者完全没登录时，卡片上拿不到价格信息，`isFreeCard` 会把所有商品误判为免费，队列被付费商品塞满后 worker 全部空跑
  - 启动时除 cookie 检查外，新增一次 `/i/users/me` 服务端 session 校验
  - "开始执行"按钮点击后会再做一次硬校验，session 失效会立即提示并阻止启动
  - worker 标签页启动前同样校验，未登录直接终止
  - 自动添加任务（`scanAndAddTasks`）在未登录态下直接跳过，避免误入队

### 性能优化

- **单任务页面就绪后 2s 无条件等待** 替换为关键元素 MutationObserver：
  - 检测到"添加到我的库 / 已保存在我的库中 / 在外部网站查看"等关键按钮出现即继续
  - 最长仍 2000ms 兜底，元素已在 DOM 上时立即跳过 → 多数任务节省 ~1–2 秒

### 验证

- 通过 `npm run build`

## 3.5.3 (2026-05-21)

### 性能优化

- **已拥有卡片隐藏延迟从 ~3.5s 降到 ~50ms**：
  - listings-states 响应拦截器在写入缓存后立即触发 `runHideOrShow()` + `checkVisibleCardsStatus()`，不再被动等 MutationObserver 兜底
  - MutationObserver 内三层 setTimeout（500+2000+1000ms）合并为单层 300ms 防抖
- **搜索/翻页/筛选请求从 +350ms 延迟降到 0**：
  - `/i/listings/search` XHR 防抖改为保守 leading-edge：距上次实际请求 ≥1.5s 时立即发出（翻页、滚动、点筛选器场景），1.5s 内仍走原 trailing 350ms 防抖
  - 两次实际请求最小间隔保持 ≥350ms，429 风险与旧版相同
  - 搜索框打字 burst 行为不变（仍合并为单次请求）

### 验证

- 通过 `npm run build`

## 3.5.2 (2026-05-12)

### 功能修复

- 修复 **免费商品不会自动进队列** 问题：
  - 自动添加遇到未加载完成的卡片时，会安排 2 秒后重试扫描
  - 列表页出现“选择许可（从 免费 到 $6.99）”这类免费许可提示时，会被视为已加载完成并自动加入待办队列
- 修复 **自动入库完成后卡片不隐藏** 问题：
  - 本地 `done` / 本会话完成记录现在会优先作为卡片已稳定状态，不再依赖列表页是否已经刷新出“已保存”文案
  - worker 成功回报写入 `sessionCompleted` 时统一使用 canonical listing URL，避免语言路径不一致导致隐藏漏判
- 修复 **已入库计数不准确** 问题：
  - 统一按 listing UID 归一化 `done` URL，避免 `/zh-cn/listings/...`、`/listings/...`、带 query 的链接重复计数
  - 启动时自动清理历史重复记录并保存，状态栏计数会随存档修正
  - 待办自动清理、DOM Refresh、任务完成写入都改用同一套去重判断
- 修复 **已拥有卡片不自动隐藏** 问题：
  - 识别新 DOM 中的“已保存在我的库中 / Saved in My Library”等文案，不再只依赖易变 class
  - DOM Refresh 确认已拥有项目后，会立即触发隐藏逻辑
  - 未加载完成的卡片不再阻塞其它已确认拥有卡片隐藏
- 修复 **日志噪声和裸 key** 问题：
  - `log_unsettled_cards` 降为 debug 日志
  - 补齐中英文文案，避免普通日志面板出现裸 key
- 修复 **隐藏付费设置不能可靠保存** 问题：
  - 补齐 `HIDE_PAID` 独立存储 key
  - 移除停止任务时一个已废弃、未定义的 `TASK` 存储 key 删除调用

### 验证

- 新增回归测试覆盖：已拥有文案识别、未加载卡片重试合并、DOM Refresh 后自动隐藏、done URL 去重、持久化 key 完整性
- 通过 `node --test tests/task-runner.test.js`
- 通过 `npm run build`

## 版本号规范

Fab Helper 脚本遵循 [语义化版本控制](https://semver.org/lang/zh-CN/) 规范：

- **主版本号**：当进行不兼容的 API 更改时增加
- **次版本号**：当添加向后兼容的新功能时增加
- **修订号**：当进行向后兼容的问题修复时增加

额外的标签：

- **alpha**：早期测试版本，功能不完整
- **beta**：功能完整但可能不稳定
- **rc**：候选发布版本，即将正式发布

例如：

- 1.0.0-alpha.1
- 1.0.0-beta.2
- 1.0.0-rc.1
- 1.0.0
